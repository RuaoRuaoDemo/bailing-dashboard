# 百灵工时数据仪表盘 使用说明

> 版本：V8.96 | 单文件 Web 应用，零后端、纯前端，浏览器打开即用

---

## 一、产品概述

百灵工时数据仪表盘是一款**纯前端**数据分析工具，面向团队管理者和组长，用于可视化展示员工的**工作量标准条**、**工时利用率**、**异常检测**和**排班管理**等核心指标。

### 核心特色

| 特色 | 说明 |
|------|------|
| 🚀 零部署 | 单个 HTML 文件，双击打开即可使用，无需安装任何服务 |
| 📊 多表融合 | 同时导入工作量表、非任务表、工时利用率底表，自动关联计算 |
| 🧮 公式中枢 | 内置标准条换算、利用率计算、异常检测等完整业务规则 |
| 💾 本地持久化 | 数据存于浏览器 localStorage，关闭再打开不丢失 |
| 📱 响应式 | 支持 PC 和移动端，自适应屏幕宽度 |
| 🎯 组织过滤 | 严格限定目标组织，自动排除非目标人员数据 |

## 二、数据导入

### 支持的数据源

打开页面后，通过顶部的导入区域加载 Excel 文件：

```
┌─────────────────────────────────────────────────┐
│  📁 工作量表(.xlsx)  │  📁 非任务表  │  📁 利用率底表  │  📁 排班表  │
└─────────────────────────────────────────────────┘
```

| 数据源 | 必需列 | 说明 |
|--------|--------|------|
| **工作量表** | 员工姓名、日期、任务名称、通航审核量、非通航任务量、任务时长(分钟)、管理效能 | 每日任务提报明细 |
| **非任务表** | 员工姓名、日期、任务名称、各绩效字段(培训/考试/开会等，单位分钟) | 非任务活动记录 |
| **利用率底表** | 员工姓名、日期、出勤时长、通航审核h、计量h、计时h、非任务h、利用率 | 工时利用率源数据 |
| **排班表** | 员工姓名、各日期列(班次编码) | 矩阵式排班数据 |

> 💡 列名自动模糊匹配，不必严格一致。导入后数据自动持久化到浏览器。

- 支持按等级筛选（严重/一般/轻微）

## 四、核心业务规则

### 4.1 标准条计算公式

| 类型 | 公式 | 说明 |
|------|------|------|
| 通航审核 | `通航量 ÷ 管理效能 × 1000` | 效能为审核量对应1000标准条的基准值 |
| 非通航审核 | `非通航量 ÷ 管理效能 × 1000` | 同上 |
| 计时任务 | `任务时长(分钟) ÷ 60 × 125` | 1小时 = 125标准条 |
| 非任务 | `非任务时长(分钟) ÷ 60 × 125` | 同计时任务 |

**日理论产出**：
- 全天班(8h) = 1000 标准条
- 半天班(4h) = 500 标准条

### 4.2 工时利用率公式

```
利用率 = ((计量h + 计时h) × 0.8125 + 通航审核h) ÷ 出勤h × 100%
```

- 通航审核有11h上限修正
- 正常区间：81.25% ~ 130%

### 4.3 异常检测规则

| 等级 | 颜色 | 利用率偏离 | 标准条偏差 |
|------|------|-----------|-----------|
| 严重 | 🔴 红 | >5% 偏离正常区间 | >5倍允差(±52条) |
| 一般 | 🟡 橙 | 2.5~5% 偏离 | 2.5~5倍允差 |
| 轻微 | 🟢 黄 | <2.5% 偏离 | <2.5倍允差 |
| 正常 | ✅ 绿 | 在正常区间内 | 在±10.42条内 |

**单日允差**：±5分钟 = ±10.42标准条

## 六、数据持久化

所有导入数据和设置存储于浏览器 `localStorage`：

| Key | 内容 |
|-----|------|
| `bailing_schedule` | 排班表数据 |
| `bailing_data_*` | 工作量/非任务/利用率表数据 |
| `bailing_settings` | 用户设置（阈值/分组规则等） |
| `bailing_inspiration_quotes` | 自定义语录 |

> ⚠️ 清除浏览器数据会丢失所有导入数据，建议定期导出备份。

## 九、适用场景

1. **日常巡检**：每日查看团队标准条产出、识别异常
2. **周期评估**：按周/月统计人均产出、利用率趋势
3. **个人画像**：深入某员工查看类目构成、历史走势
4. **排班管理**：维护班表、自动识别半天假/全天假出勤折算
5. **异常追踪**：实时检测数据异常、工时异常，辅助管理决策

---

## 十、注意事项

1. **浏览器要求**：推荐 Chrome/Edge 最新版，需开启 JavaScript
2. **数据安全**：所有数据仅存于本地浏览器，不上传任何服务器
3. **缓存刷新**：如排班数据更新后显示异常，可尝试重新导入排班表
4. **组织过滤**：仅处理目标组织数据（云雀/第二交付中心/二项目组/A组/H1组 和 嘉兰项目）
5. **大文件**：支持约50万行级别底表数据，首次加载可能需数秒
